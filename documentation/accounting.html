Accounting add-on

About

The Accounting add-on extends the functionality of Crypto Trading Competitions and Stock Trading Competitions applications and provides the following features.

Accounts

An account is automatically created for each user to track movements of real funds. Current balance as well as account transactions history is displayed on the user Account page. A website administrator can set any account currency (to be used globally by all accounts).

Deposits

Users can deposit real money to their accounts using PayPal, Stripe and CoinPayments.net payment gateways. The following payment methods are supported by Stripe integration:

Debit / Credit card
IDEAL
Bancontact
Giropay
SOFORT
EPS
Multibanco
Przelewy24
Alipay

An administrator can choose which payment methods are available to users.

Withdrawals

Users can withdraw funds from their accounts. Withdrawal forms for PayPal and wire transfer methods are provided, but payments will need to be executed manually by administrators. Once a payment is marked as Completed in the backend the user account balance is automatically decreased. An administrator can choose which withdrawal methods are available to users.

Coinpayments.net

Please follow next steps to configure payments via coinpayments.net:

  • Sign up at coinpayments.net.
  • Open Account » Coin Acceptance Settings page and select coins that you would like to accept (these coins will be available for deposits and withdrawals).
  • Open Account » Account Settings page, select Merchant Settings tab and fill IPN Secret field. It can be any random alphanumeric string. Do not fill IPN URL, leave it blank.
  • Open Account » API keys page and click Generate new key button. After it is generated click Edit Permissions button and enable the following permissions:
    • get_basic_info
    • create_transaction
    • get_tx_info
    • balances
    • create_withdrawal
    • Allow auto_confirm = 1 in create_withdrawal (only in case you like to enable auto confirmation of withdrawals)
    • get_withdrawal_info
    • rates
  • Copy Merchant ID, Public key, Private key and IPN Secret. Input these keys in Backend » Settings » Add-on: Accounting. Add Coinpayments.net to Enabled deposit methods.
If you use a firewall or some external security service such as Cloudflare make sure to white list all GET / POST requests to all URLs that start with /deposits, otherwise instant payment notification (IPN) messages will not reach your website and hence deposits / withdrawals will stuck in Pending status.

Installation

Step 1

Log in to the backend and make sure the minimum required version is installed (the version is displayed in the top header bar), open Maintenance page and leave it open.

Step 2

Download the latest application package from CodeCanyon, upload it to your server and unzip (overwrite all existing files), so that such folders as app, bootstrap, config etc are on the same level with packages folder.

Step 3

Go back to Maintenance page (or open the following URL: https://yourwebsite.com/admin/maintenance), click Run database updates button and after this process is successfully completed click Clear cache button.

Step 4

Open Add-ons page in the backend and check that the add-on is marked as installed.

Step 5

Open Settings page in the backend and under Add-on: Accounting tab configure the add-on settings, such as account currency, PayPal API credentials, Stripe API credentials, allowed payment and withdrawal methods.

Upgrade

To upgrade the add-on to the latest version simply repeat the installation steps.

Support

Please note that support can only be provided during the application support period. 6 months of free support are provided with your purchase. Should you need support outside of this period you will need to renew it. How to extend / renew the app support?

To get technical support please submit a new ticket at https://support.financialplugins.com. If you see an application error please do the following:

  • Enable Debug mode (Backend » Settings » Developer)1
  • Repeat the error
  • Make a screenshot of the error
  • Check the browser console and make a screenshot if there are any errors
  • Provide the above information along with the application error log (can be found in storage/logs folder)
  • Provide steps to reproduce the error

In case the issue is difficult to trace our support team can request SSH / cPanel access to your server and / or website admin access.

1 — If you don't have access to backend please edit .env configuration file (located in the web root folder of your domain) and set the following variables: 

APP_DEBUG=true
APP_LOG_LEVEL=debug
