Cash competitions add-on

About

The Cash competitions add-on extends the functionality of Crypto Trading Competitions and Stock Trading Competitions applications and provides the following features.

Fees

An administrator can create trading competitions, which require users to pay a predefined fee before joining such competition. Fees are deducted from user account balance, so users will need to deposit funds before fees can be paid.

Rewards (cash prizes)

An administrator can set up automatic allocation of collected fees between competition winners. It's possible to set either percentage allocation (for example, 30% of total fees collected goes to the winner, 20% of total fees goes to the runner-up and 10% of total fees goes to the third-place finisher) or a flat reward (for example, 50 USD paid to the winner, 30 USD paid to the runner-up, 10 USD paid to the third-place finisher). Winners can then spend their cash rewards on other paid competitions or withdraw them.

Installation

Step 1

Log in to the backend and make sure the minimum required version is installed (the version is displayed in the top header bar), open Maintenance page and leave it open.

Step 2

Download the latest application package from CodeCanyon, upload it to your server and unzip (overwrite all existing files), so that such folders as app, bootstrap, config etc are on the same level with packages folder.

Step 3

Go back to Maintenance page (or open the following URL: https://yourwebsite.com/admin/maintenance), click Run database updates button and after this process is successfully completed click Clear cache button.

Step 4

Open Add-ons page in the backend and check that the add-on is marked as installed.

Upgrade

To upgrade the add-on to the latest version simply repeat the installation steps.

Support

To get technical support please submit a new ticket at https://support.financialplugins.com. If you see an application error please do the following:

  • Enable Debug mode (Backend » Settings » Developer)1
  • Repeat the error
  • Make a screenshot of the error
  • Check the browser console and make a screenshot if there are any errors
  • Provide the above information along with the application error log (can be found in storage/logs folder)
  • Provide steps to reproduce the error

In case the issue is difficult to trace our support team can request SSH / cPanel access to your server and / or website admin access.

1 — If you don't have access to backend please edit .env configuration file (located in the web root folder of your domain) and set the following variables: 

APP_DEBUG=true
APP_LOG_LEVEL=debug
